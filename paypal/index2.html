The PayPal Mini Cart

The PayPal Mini Cart is a great way to improve your PayPal Cart integration by creating an overlay on your e-commerce website which appears as a user adds a product to their cart. This results in a smoother user experience for the customers and they never need to leave your website until they're ready to checkout!

Try it out below by adding a product to your cart:

Product 1

  • Shipping: $1.00
  • Price: $1.00

Product 2

  • Item Number: 12345
  • Price: $2.00

Product 3

Note: Proceeding to checkout will bring you to the live PayPal website. Please do not complete the transaction.

Interested? Let's get you setup

  1. Make sure you have a PayPal Shopping Cart enabled page on your website. If you don't you can learn how to set one up over at our Developer Central.
  2. Download and save the Mini Cart JavaScript (version 1.4). There's also a minified version available.
  3. Next, you'll need to include the following bit of JavaScript on your page before the closing </body> tag (make sure to change the file path to where you saved the JavaScript file): 
    <script src="/path/to/minicart.js" type="text/javascript"></script>
  4. Finally, send a note to labs-feedback-minicart@paypal.com and let us know what you think.

It's as simple as that! Now the Mini Cart will appear when a user adds a product to, views, or has an item in their cart.

Customization

You can customize the script to make it work for your site by creating a JavaScript object, MiniCart, and setting any of the following optional properties.

displayEdge
The edge of the page the cart should pin to. Set to "L" for left or "R" for right.
edgeDistance
Distance from the edge the cart should appear.
cookiePath
The base path of your website to set the cookie to (primarily for shared website hosting).
currencySymbol
The symbol which appears before the price and subtotal.
onAddToCart
A custom function run before a product is added to the cart. The product is passed as the argument. Returning a value of false will cancel adding the product.
onUpdateTotal
A custom function run before the total is updated. Available variables are this.products and this.settings.
onCheckout
A custom function run before the cart contents are sent to PayPal. Available variables are this.products and this.settings. Returning a value of false will cancel checkout.

See below for an example:

<script type="text/javascript">

	    var MiniCart = {

	        displayEdge: 'R',


	        edgeDistance: '50px',

	        cookiePath: '/',

	        currencySymbol: '$',

	        onAddToCart: function (product) { ... },


	        onUpdateTotal: function () { ... },

	        onCheckout: function () { ... }

	    };


 </script>

FAQ

  1. Are hosted / encrypted buttons supported?
    The Mini Cart doesn't yet work with what we call our "hosted" or "encrypted" buttons. We're working on support for them and will release an update once it's available.
    • If you are logged in to paypal.com and are creating or already have a hosted / encrypted button and you'd like to use the Mini Cart you'll need to:
      1. Create a button on PayPal's website and uncheck the Save button at PayPal checkbox under Step 2: Track inventory, profit & loss.
      2. Once you've created the button click Remove code protection before copying your button's code.
      This will provide you with button code you can include on your website and works with the Mini Cart.
    • If you are not logged in when you create a button you shouldn't need to do any additional steps to be compatible with the Mini Cart.

    Note: If using a non-hosted button you must check that your payment was for the correct amount before delivering any goods.

  2. Is non-JavaScript supported?
    If your users do not have a JavaScript-capable browser, they will still be able to see your cart buttons and make purchases, but the user interface will gracefully degrade to the standard PayPal cart.
  3. The Mini Cart isn't appearing the same as on this page. Why?
    This can occur if your page is being rendered in the browser's Quirks mode. Example issues include being permanently displayed in the bottom left of the screen or not scrolling properly. To fix the issue you can validate and correct your HTML markup using the W3C Markup Validator.
